# 코드 최적화 태스크 포스 (Code Optimization Task Force)

이 스쿼드는 4단계 순차적 워크플로우를 사용하여 코드베이스를 리팩토링하고, 성능을 개선하며, 보안을 강화하는 데 집중합니다.

## 에이전트
- **Opt_Architect (아키텍트)**: 구조적 분해, 모듈화, SRP/DRY 원칙 준수 담당.
- **Opt_Performance (성능)**: 속도, 메모리 사용량, 알고리즘 효율성 담당.
- **Opt_Cleaner (클리너)**: 불필요한 코드 제거, 복잡성 단순화 담당.
- **Opt_Security (보안)**: 에러 처리, 취약점 패치, 타입 안전성 담당.

## 순차적 워크플로우
1. **정리 (Clean)**: `Opt_Cleaner`를 활성화하여 불필요한 요소를 제거합니다.
2. **구조화 (Structure)**: `Opt_Architect`를 활성화하여 로직과 흐름을 개선합니다.
3. **최적화 (Optimize)**: `Opt_Performance`를 활성화하여 성능을 향상시킵니다.
4. **보안 (Secure)**: `Opt_Security`를 활성화하여 안전성을 강화합니다.
